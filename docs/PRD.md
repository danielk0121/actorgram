# PRD — actors

## 공통

- 앱 화면의 모든 정보는 한글로 표기한다.
- 최상단 메뉴바를 가진다. 앱 이름과 파비콘은 좌측, 메뉴는 우측에 정렬한다.
- 좌측 로고(앱 이름 또는 파비콘)를 클릭하면 홈 화면(배우 검색 화면)으로 이동한다.
- 홈 화면은 배우 검색 화면과 동일하며, 상단바에 별도 홈 메뉴를 두지 않는다.
- 메뉴 항목: 배우, 영화, 사진 검색, AI 검색

## 배우 검색 화면

- 배우 이름 또는 배역 이름으로 배우를 검색할 수 있다.
- 검색어가 없으면 전체 배우 목록을 표시한다.
- 검색 결과 섹션 제목에 전체/검색 배우 수를 표시한다. (예: "전체 배우 (42)", "검색 배우 (3)")
- 배우 목록은 배역 이미지 수 기준 내림차순으로 정렬된다.

### 배우 카드

- 배우 카드는 상단 배우 정보 영역과 하단 출연 정보 카드 목록으로 구성된다.
- 배우 정보 영역: 프로필 이미지, 이름, 출생연도, 국적, 성별, 데뷔일, 출연 영화 수를 표시한다.
- 배우 정보 영역 클릭 시 배우 상세 화면으로 이동한다.
- 출연 정보 카드: 배역 이미지가 있는 영화를 최근 연도 순으로 최대 3개 표시한다.
- 출연 정보 카드 각각에는 영화 포스터, 영화 제목, 개봉연도, 배역명, 배역 이미지 썸네일 3장을 표시한다.
- 출연 정보 카드 클릭 시 영화 상세 화면으로 이동한다.

## 배우 상세 화면

- 배우 기본 정보를 표시한다: 프로필 이미지, 이름, 출생연도, 국적, 성별, 데뷔일
- 영화 속 이미지 섹션을 표시한다. 섹션 제목에 전체 이미지 수와 영화 수를 표시한다.
- 영화 제목으로 영화를 검색할 수 있다.
- 영화 목록을 연도순/영화제목순으로 정렬할 수 있다. 각 정렬 기준은 오름차순/내림차순 토글 버튼으로 전환한다.
- 기본 정렬은 연도 내림차순이다.
- 영화별로 배역 이미지를 그리드로 표시한다. 최대 9장을 표시하며, 초과 시 마지막 칸에 "+N" 오버레이를 표시한다.
- 영화 행 클릭 시 영화 상세 화면으로 이동한다.

## 영화 검색 화면

- 영화 제목, 장르, 감독으로 영화를 검색할 수 있다.
- 검색어가 없으면 전체 영화 목록을 표시한다.
- 영화 목록은 1줄에 1개씩 표시되며, 포스터와 정보가 가로로 나란히 배치된다.
- 영화 카드에는 포스터, 제목, 연도, 장르, 제작국가, 감독, 작가, 관람등급, 상영시간, 개봉일을 표시한다.
- 영화 카드에 해당 영화의 주연배우 전체를 표시한다. 각 주연배우에는 프로필 이미지, 이름, 배역명을 표시한다.
- 주연배우 클릭 시 배우 상세 화면으로 이동한다.
- 영화 카드 클릭 시 영화 상세 화면으로 이동한다.

## 영화 상세 화면

- 영화 기본 정보를 표시한다: 포스터, 제목, 연도, 장르, 감독, 작가, 관람등급, 상영시간, 개봉일, 제작국가
- 줄거리 섹션을 표시한다.
- 주연배우 섹션을 표시한다. 각 배우에는 프로필 이미지, 이름, 배역명, 배역 이미지 3장을 표시한다.
- 주연배우 클릭 시 배우 상세 화면으로 이동한다.

## 사진 검색 화면

- 사진을 드래그하거나 클릭하여 업로드할 수 있다.
- 사진 업로드 시 샘플 검색 결과를 표시한다. (현재 샘플: 톰 크루즈, 맷 데이먼)
- 샘플 결과는 실제 API 연동 후 인식된 배우 기준 결과로 교체될 예정이다.

## 유비쿼터스 언어 (용어 사전)

도메인 전반에서 일관되게 사용하는 용어를 정의한다. 코드, PRD, 커뮤니케이션 모두 이 용어를 기준으로 한다.

### 데이터 구조 관계도

```
Movie (작품)
├── cast: CastEntry[]          ← 출연진 (CastEntry의 집합)
│   └── CastEntry (출연 정보)  ← 배우 1명 × 작품 1편의 출연 기록
│       ├── actorId            → Actor 참조
│       ├── role               ← 배역명 (문자열)
│       └── roleImages[]       ← 배역 이미지 목록
└── mainActors: number[]       ← 주연배우 actorId 목록 (CastEntry 중 주연만)

Actor (배우)
├── id
├── name
└── imageUrl                   ← 프로필 이미지

필모그래피 = 특정 Actor가 출연한 CastEntry 전체 목록 (Movie를 가로질러 집계)
```

### CastEntry 집합의 두 가지 관점

> 영화 1개 관점에서 배우들 집합 > cast > 각 요소는 castEntry
> 배우 1명 관점에서 영화 배역 정보 집합 > 필모그래피 > 각 요소는 castEntry

같은 CastEntry를 어느 관점에서 바라보느냐에 따라 집합 이름만 달라진다. 원소(CastEntry)는 동일하다.

```
영화 관점:  Movie.cast: CastEntry[]        ← 이 영화에 출연한 모든 배우의 출연 기록
배우 관점:  Actor의 필모그래피: CastEntry[] ← 이 배우가 출연한 모든 영화의 출연 기록
```

- `Movie.cast` → 소스코드에 실체(배열 필드)가 있음
- `필모그래피` → 소스코드에 실체 없음, 필요할 때 `SAMPLE_MOVIES.filter(...)`로 동적 집계

| 관점 | 집합 이름 | 소스코드 | 비고 |
|---|---|---|---|
| 영화 1개 → 출연한 배우들 | 출연진 (Cast) | `Movie.cast: CastEntry[]` | 소스코드에 배열 필드로 실체 있음 |
| 배우 1명 → 출연한 영화들 | 필모그래피 (Filmography) | — | 소스코드에 실체 없음. 필요 시 동적 집계 |

### 용어 정의

| 한글 | 영어 | 소스코드 | 설명 |
|---|---|---|---|
| 배우 | Actor | `Actor` | 영화 또는 작품에 출연한 인물. 애니메이션의 경우 성우를 배우로 취급한다. |
| 작품 | Movie | `Movie` | 영화, 애니메이션 등 배우가 출연하는 콘텐츠 단위. |
| 출연진 | Cast | `cast` | 작품에 출연한 CastEntry의 집합. `Movie.cast: CastEntry[]`. |
| 출연 정보 | Cast Entry | `CastEntry` | 배우 1명 × 작품 1편의 출연 기록 1건. 출연진(cast)의 원소. 배역명과 배역 이미지를 포함한다. |
| 주연배우 | Main Cast | `mainActors` | 작품의 주연 Actor id 목록. 배역(CastEntry)이 아닌 배우(Actor) 단위로 주연 여부를 결정한다. |
| 배역 | Role | `role` | 배우가 작품에서 맡은 등장인물 이름. CastEntry의 속성. |
| 필모그래피 | Filmography | — | 특정 배우의 전체 출연 기록. 해당 Actor의 CastEntry를 작품을 가로질러 집계한 목록. |
| 프로필 이미지 | Profile Image | `imageUrl` | 배우의 대표 사진. Actor의 속성. |
| 배역 이미지 | Role Image | `roleImages` | 배우가 특정 작품에서 배역을 연기하는 장면 이미지 목록. CastEntry의 속성. |
| 배우 카드 | Actor Card | `ActorCard` | 배우 검색 결과에서 배우 1명의 정보와 출연 정보 카드 목록을 묶어 표시하는 UI 단위. |
| 출연 정보 카드 | Cast Entry Card | `CastEntryCard` | 배우 카드 안에서 CastEntry 1건의 정보(포스터·제목·개봉연도·배역명·배역 이미지 썸네일)를 표시하는 UI 단위. |
| 작품 형식 | Format | `format` | 작품이 단편인지 시리즈인지 구분하는 속성. `'단편'` 또는 `'시리즈'`. |
| 에피소드 | Episode | `episode` | 시리즈 작품의 총 에피소드 수. |
